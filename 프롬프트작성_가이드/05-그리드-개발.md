# 그리드 개발 프롬프트 예제

> DataGrid, GroupGrid, TreeGrid의 CRUD, 셀 서식, 이벤트 처리 등을 구현할 때 사용합니다.

---

## 핵심 포인트

- **그리드 타입**을 명시 (DataGrid / GroupGrid / TreeGrid)
- **컬럼 구성**과 **편집 가능 여부**를 설명
- **CRUD 패턴**이 필요한지 명시 (행 추가/삭제/수정/저장)
- 관련 Skill: `/iaud-grid-guide`

---

## 예제 1: 기본 DataGrid 조회 + 편집

```
OrderMgmt.script.ts에서 GRD_MAIN (DataGrid)에 대해:

<컬럼 구성>
- ORDER_ID: 텍스트, 읽기 전용
- ORDER_NAME: 텍스트, 편집 가능
- STATUS: 콤보박스 (DS_STATUS 바인딩, CODE/NAME)
- AMOUNT: 숫자, 편집 가능, "#,##0" 포맷
- ORDER_DATE: 날짜, Calendar 컬럼

<기능 구현>
1. 조회: BTN_SEARCH 클릭 시 DS_ORDERS 데이터소스 조회
2. 행 추가: BTN_ADD 클릭 시 마지막 행에 빈 행 추가
3. 행 삭제: BTN_DEL 클릭 시 선택 행 삭제 마킹
4. 저장: BTN_SAVE 클릭 시 서버 SaveOrders 호출
5. 조회 완료 시 첫 번째 행 자동 선택

/iaud-grid-guide 참조
```

## 예제 2: GroupGrid 소계/합계

```
GRD_SALES (GroupGrid) 설정을 수정해줘.

<현재 상태>
- .design.json에 GroupGrid로 정의되어 있음
- 그룹 키 컬럼: REGION (지역), MONTH (월)

<원하는 동작>
- 지역별 소계 표시 (소계 위치: 하단)
- 전체 총합계 표시 (총합계 위치: 상단)
- AMOUNT 컬럼: 소계/총합계에 Sum 계산
- QUANTITY 컬럼: 소계/총합계에 Sum 계산
- UNIT_PRICE 컬럼: 소계/총합계에 None (빈칸)
- 소계 행 배경색: #F5F5F5
- 총합계 행 배경색: #E3F2FD
- 소계 텍스트: "{그룹값} 소계"
- 총합계 텍스트: "전체 합계"

/iaud-grid-guide 참조해서 GroupGrid 설정 적용
```

## 예제 3: TreeGrid 계층 구조

```
GRD_DEPT (TreeGrid) 를 구현해줘.

<데이터 구조>
- DEPT_CODE: 부서 코드 (PK)
- PARENT_CODE: 상위 부서 코드 (FK, 최상위는 빈 값)
- DEPT_NAME: 부서명
- DEPT_LEVEL: 깊이 (1, 2, 3)
- MANAGER: 부서장 이름

<기능>
- 트리 구조로 부서 계층 표시
- 초기 로드 시 1레벨만 펼침
- BTN_EXPAND_ALL: 전체 펼치기
- BTN_COLLAPSE_ALL: 전체 접기
- 행 클릭 시 우측 상세 영역에 부서 정보 표시

/iaud-grid-guide 참조
```

## 예제 5: 그리드 데이터 서버 전송 (CRUD)

```
GRD_EMP (DataGrid)의 변경 데이터를 서버로 전송하는 코드를 작성해줘.

<클라이언트 측>
- BTN_SAVE 클릭 시 GRD_EMP의 CRUD 데이터를 서버로 전송
- Matrix.RunScriptEx 사용
- gridNames 파라미터에 "GRD_EMP" 지정하여 자동 데이터 바인딩

<서버 측>
- ServerScript/SaveEmp.ts 에서 DataTable로 수신
- __CRUD__ 필드로 C(신규)/U(수정)/D(삭제) 구분
- 트랜잭션 내에서 DELETE → UPDATE → INSERT 순서 처리

/iaud-grid-guide 참조해서 CRUD 전체 패턴 구현
/iaud-server-script 참조
```

## 그리드 관련 자주 하는 실수

```
그리드 코드 작성 시 아래 사항에 주의해줘:

1. Matrix.getObject("GRD_XXX") 결과는 반드시 `as DataGrid` 등 구체적 타입으로 캐스트
2. args.Row 직접 사용 (별도 변수에 할당 불필요)
3. 행 삭제는 물리 삭제가 아닌 CRUD 마킹 (row.__CRUD__ = "D")
4. 컬럼 편집 가능/불가는 .design.json의 ColumnType으로 설정
5. 행 추가 후 추가된 행으로 스크롤 이동
```

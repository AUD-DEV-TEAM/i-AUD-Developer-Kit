# 클라이언트 스크립트 개발 프롬프트 예제

> `.script.ts` 파일에서 UI 이벤트 처리, 컨트롤 조작, 서버 호출 등을 구현할 때 사용합니다.

---

## 핵심 포인트

- **컨트롤 이름(Name)과 타입**을 정확히 명시
- **이벤트명**을 알려주면 더 정확 (OnClick, OnCellClick, OnTextKeydown 등)
- **서버 호출**이 필요하면 서비스명과 파라미터도 명시
- 관련 Skill: `/iaud-client-script`

---

## 예제 1: 버튼 클릭 → 그리드 조회

```
src/reports/Work/OrderMgmt/OrderMgmt.script.ts 수정해줘.

BTN_SEARCH (Button) 클릭 시:
1. TXT_FROM_DATE (TextBox)와 TXT_TO_DATE (TextBox)의 값을 가져와서
2. DS_ORDERS 데이터소스의 파라미터로 설정
3. GRD_MAIN (DataGrid)에 조회 실행
4. 조회 완료 후 상태바에 "N건 조회됨" 메시지 표시

/iaud-client-script 참조
```

## 예제 2: 그리드 셀 클릭 이벤트

```
OrderMgmt.script.ts에서 GRD_MAIN (DataGrid) 셀 클릭 이벤트를 추가해줘.

- OnCellClick 이벤트에서:
  - 클릭한 행의 ORDER_ID 값을 LBL_ORDER_ID (Label)에 표시
  - 클릭한 행의 ORDER_STATUS 값에 따라:
    - "PENDING": BTN_APPROVE 활성화, BTN_CANCEL 활성화
    - "APPROVED": BTN_APPROVE 비활성화, BTN_CANCEL 활성화
    - "CANCELLED": 둘 다 비활성화
  - 스타일 변경은 컨트롤 API 사용 (Element 직접 접근 금지)
```

## 예제 3: 서버 스크립트 호출 (RunScriptEx)

```
OrderMgmt.script.ts에서 BTN_SAVE (Button) 클릭 시 서버에 데이터를 저장해줘.

- GRD_MAIN (DataGrid)의 변경된 데이터를 서버로 전송
- 서버 스크립트명: "SaveOrders"
- 파라미터: { userId: Matrix.UserID }
- 성공 시: "저장되었습니다" 알림 후 재조회
- 실패 시: 서버 에러 메시지 표시

/iaud-client-script 참조해서 RunScriptEx 패턴 적용
```

## 예제 4: 조건부 컨트롤 표시/숨김

```
OrderMgmt.script.ts에서 CHK_DETAIL (CheckBox) 체크 여부에 따라
PNL_DETAIL (Panel) 영역을 표시/숨김 처리해줘.

- CHK_DETAIL 체크 시: PNL_DETAIL.Visible = true
- CHK_DETAIL 해제 시: PNL_DETAIL.Visible = false
- Visible 변경 후 Matrix.Resize() 호출하여 레이아웃 재계산
```

## 예제 5: 팝업 보고서 열기/결과 받기

```
OrderMgmt.script.ts에서 BTN_POPUP (Button) 클릭 시
팝업 보고서를 열고 결과를 받아와줘.

- 팝업 보고서 코드: "RPT_CUSTOMER_SEARCH"
- 전달 파라미터: { keyword: TXT_KEYWORD.Text }
- 팝업에서 선택 후 돌아오면:
  - 결과의 CUST_CODE를 TXT_CUST_CODE에 설정
  - 결과의 CUST_NAME을 TXT_CUST_NAME에 설정

Matrix.ShowModalReport 또는 Matrix.OpenReport 사용
```

## 예제 6: TextBox Enter 키 이벤트

```
OrderMgmt.script.ts에서 TXT_KEYWORD (TextBox)에 Enter 키 입력 시
조회 함수를 호출해줘.

- OnTextKeydown 이벤트 사용
- args.Event.isEnter()로 Enter 키 판별
- 조회 함수(doSearch)를 별도로 분리하여 버튼 클릭과 Enter 키 양쪽에서 호출
```

## 예제 7: 동적 스타일 변경

```
OrderMgmt.script.ts에서 GRD_MAIN의 데이터 로드 완료 후
STATUS 컬럼 값에 따라 행 배경색을 다르게 표시해줘.

- "COMPLETE": 연한 녹색 (#E8F5E9)
- "ERROR": 연한 빨간색 (#FFEBEE)
- "PENDING": 기본색 유지

주의: 반드시 컨트롤 스타일 API 사용, Element.style 직접 접근 금지
/iaud-grid-guide 참조
```

## 주의사항 체크리스트

프롬프트 작성 시 아래 규칙을 인지하고 있으면 AI 결과가 더 정확해집니다:

- [ ] `Matrix.getObject()` 결과는 구체적 타입으로 캐스트 (`as Button`, `as DataGrid` 등, `as any` 금지)
- [ ] AddIn/ExternalComponent는 `OnComponentClassLoaded` 내에서만 접근
- [ ] 스타일 변경은 컨트롤 API 사용 (`.Style.Background.Color.SetColor()` 등)
- [ ] `innerHTML` 사용 금지 (보안 검출 대상)
- [ ] Constructor 함수는 class로 변환 금지

# 새 보고서 생성 프롬프트 예제

> 보고서를 처음부터 새로 만들 때 사용하는 프롬프트 예제입니다.

---

## 핵심 포인트

- **보고서 이름, 코드, 경로**를 명확히 지정
- **필요한 컨트롤 목록**과 **배치(레이아웃)**를 설명
- **DataSource(SQL)**와 **ServerScript**가 필요한지 명시
- MCP 도구가 있으면 `/iaud-mtsd-create` Skill 활용 가능

---

## 예제 1: 기본 조회 보고서

```
새 보고서를 만들어줘.

<보고서 정보>
- 보고서명: 주문관리
- 폴더 경로: src/reports/Work/OrderMgmt/
- 보고서 코드: RPT_ORDER_MGMT

<화면 구성>
- 상단 영역 (높이 40px, 좌우 도킹):
  - TXT_KEYWORD (TextBox): 검색어 입력, 너비 200px
  - BTN_SEARCH (Button): "조회" 버튼, 너비 80px
- 하단 영역 (나머지 전체):
  - GRD_MAIN (DataGrid): 주문 목록 그리드, 상하좌우 도킹

<DataSource>
- DS_ORDERS: 주문 조회 SQL (AUD_SAMPLE_DB 연결)
  - 파라미터: @KEYWORD (TXT_KEYWORD 바인딩)

<동작>
- BTN_SEARCH 클릭 시 GRD_MAIN에 DS_ORDERS 조회
- TXT_KEYWORD에서 Enter 입력 시에도 조회 실행
- 조회 전 "조회 중..." 메시지 표시

/iaud-mtsd-create 참조해서 .design.json 생성해줘
```

## 예제 2: CRUD 보고서 (입력/수정/삭제 포함)

```
CRUD 기능이 있는 보고서를 만들어줘.

<보고서 정보>
- 보고서명: 사원관리
- 폴더 경로: src/reports/Work/EmpMgmt/
- 보고서 코드: RPT_EMP_MGMT

<화면 구성>
- 상단 툴바 (높이 40px, 좌우 도킹):
  - BTN_SEARCH (Button): "조회"
  - BTN_ADD (Button): "행 추가"
  - BTN_DELETE (Button): "행 삭제"
  - BTN_SAVE (Button): "저장"
- 하단 그리드 (나머지 전체, 상하좌우 도킹):
  - GRD_EMP (DataGrid, 편집 가능):
    - EMP_NO (텍스트, 신규 시 입력 가능, 수정 시 읽기 전용)
    - EMP_NAME (텍스트, 입력 필드)
    - DEPT_CODE (콤보박스, DS_DEPT 바인딩)
    - SALARY (숫자, 입력 필드, 천단위 콤마)
    - HIRE_DATE (날짜, Calendar 컬럼)

<DataSource>
- DS_EMP: 사원 조회 SQL
- DS_DEPT: 부서 코드 SQL (콤보박스용)

<서버 스크립트>
- SaveEmp: GRD_EMP의 CRUD 데이터를 받아 INSERT/UPDATE/DELETE 처리

<동작>
- BTN_ADD: 그리드 마지막 행에 빈 행 추가
- BTN_DELETE: 선택된 행 삭제 마킹
- BTN_SAVE: Matrix.RunScriptEx로 SaveEmp 호출, 성공 시 재조회

/iaud-grid-guide 참조해서 CRUD 패턴 적용해줘
/iaud-server-script 참조해서 서버 스크립트 작성해줘
```

## 예제 3: 탭 기반 마스터-디테일 보고서

```
마스터-디테일 구조의 보고서를 만들어줘.

<보고서 정보>
- 보고서명: 프로젝트관리
- 폴더 경로: src/reports/Work/ProjectMgmt/
- 보고서 코드: RPT_PROJECT_MGMT

<화면 구성>
- 좌측 (너비 300px, 상하 도킹):
  - TXT_SEARCH (TextBox): 프로젝트 검색
  - GRD_PROJECT (DataGrid): 프로젝트 목록, 읽기 전용
- 우측 (나머지 전체, 상하 도킹):
  - TAB_DETAIL (TabControl):
    - 탭1 "기본정보": Label + TextBox 폼 레이아웃
    - 탭2 "작업목록": GRD_TASK (DataGrid, 편집 가능)
    - 탭3 "이력": GRD_HISTORY (DataGrid, 읽기 전용)

<동작>
- GRD_PROJECT 행 클릭 시 선택된 프로젝트의 상세 정보를 우측 탭에 표시
- 각 탭은 선택된 프로젝트 코드를 기준으로 데이터 조회
```

## 예제 4: 최소 정보로 요청 (AI가 추론)

```
간단한 코드 조회 보고서를 만들어줘.
- 폴더: src/reports/Work/CodeMgmt/
- 코드 그룹 / 코드 상세 마스터-디테일 구조
- 좌측에 코드 그룹 그리드, 우측에 코드 상세 그리드
- 양쪽 그리드 모두 편집/저장 가능
- /iaud-mtsd-create 참조
```

> **팁**: 최소 정보로도 요청 가능하지만, 컨트롤 이름과 컬럼 구성을 명시할수록 결과가 정확합니다.

---

## 주의사항

1. **보고서 코드**는 서버에 등록된 유일한 식별자이므로 기존 코드와 중복되지 않도록 주의
2. **폴더 경로**는 반드시 `src/reports/` 하위에 위치
3. `.design.json` 생성 후 VS Code 확장의 `AUD: Push Report` 명령으로 서버에 업로드
4. DataSource SQL에서 테이블명은 실제 DB 스키마에 맞게 작성

# 서버 스크립트 개발 프롬프트 예제

> `ServerScript/` 폴더의 `.ts` 파일에서 DB 조회, 비즈니스 로직, 파일 처리 등을 구현할 때 사용합니다.

---

## 핵심 포인트

- **서버 스크립트는 Rhino(ES5) 엔진에서 실행** — ES2015+ 문법 사용 불가
- **DB 연결 코드(Connection Code)**를 명시해야 AI가 올바른 연결 생성
- **입력/출력 데이터 구조**를 설명하면 정확한 코드 생성 가능
- 관련 Skill: `/iaud-server-script`

---

## 예제 1: 기본 DB 조회 서비스

```
src/reports/Work/OrderMgmt/ServerScript/GetOrders.ts 를 만들어줘.

<서비스 설명>
- 클라이언트에서 RunScriptEx("", "GetOrders", params, callback) 으로 호출
- DB 연결 코드: "AUD_SAMPLE_DB"

<입력 파라미터>
- fromDate: 시작일 (YYYYMMDD)
- toDate: 종료일 (YYYYMMDD)
- keyword: 검색어 (선택, 빈 값일 수 있음)

<처리 로직>
- 주문 테이블(TB_ORDER)에서 ORDER_DATE 범위 조회
- keyword가 있으면 ORDER_NAME LIKE 조건 추가
- 결과를 DataSet으로 리턴

<출력>
- DataTable "orders": ORDER_ID, ORDER_NAME, ORDER_DATE, STATUS, AMOUNT

/iaud-server-script 참조
```

## 예제 2: CRUD 저장 서비스 (Insert/Update/Delete)

```
ServerScript/SaveOrders.ts 를 만들어줘.

<서비스 설명>
- 클라이언트에서 GRD_MAIN의 변경 데이터를 CRUD 방식으로 전송
- DB 연결 코드: "AUD_SAMPLE_DB"

<테이블 정보>
- 테이블: TB_ORDER
- PK: ORDER_ID
- 컬럼: ORDER_ID, ORDER_NAME, CUST_CODE, ORDER_DATE, AMOUNT, STATUS

<처리 순서>
1. 트랜잭션 시작
2. 삭제(D) → 수정(U) → 신규(C) 순서로 처리
3. 각 행의 __CRUD__ 필드로 C/U/D 구분
4. 전체 성공 시 커밋, 오류 시 롤백

/iaud-server-script 참조해서 ScriptConnection 트랜잭션 패턴 적용
/iaud-grid-guide 참조해서 CRUD 데이터 수신 패턴 적용
```

```
/iaud-server-script 참조
```

## 예제 4: 외부 API 호출

```
ServerScript/CallExternalAPI.ts 를 만들어줘.

<서비스 설명>
- 외부 REST API를 호출하여 결과를 DataTable로 변환

1. ScriptHttpClient로 GET 요청 전송

3. DataTable로 변환하여 리턴

/iaud-server-script 참조
```

## 예제 5: 공통 모듈 (@로 시작하는 서버 스크립트)

```
ServerScript/@CommonUtils.ts 공통 모듈을 만들어줘.

<설명>
- 여러 서비스에서 공통으로 사용하는 유틸 함수 모음
- @로 시작하는 파일은 자동 로드되는 공통 모듈

<함수 목록>
- formatDate(date: string, format: string): 날짜 포맷 변환
- getConnectionCode(): 환경별 DB 연결 코드 반환
- logAction(userId, action, detail): 감사 로그 INSERT

주의: Rhino 환경이므로 `in` 연산자 사용 금지, hasOwnProperty 사용
```

## Rhino 제약사항 체크리스트

프롬프트에 아래 내용을 AI에게 상기시키면 더 안전한 코드가 생성됩니다:

```
서버 스크립트 작성 시 Rhino(ES5) 제약사항을 반드시 준수해줘:
- `in` 연산자 사용 금지 → obj.key !== undefined 또는 obj.hasOwnProperty("key") 사용
- Array.from(), Object.entries(), Object.keys() 사용 금지
- Map, Set, Promise, async/await 사용 금지
- 화살표 함수(=>) 사용 금지 → function() {} 사용
- let/const 사용 금지 → var 사용 (tsc가 ES5로 변환하므로 .ts에서는 let/const 사용 가능)
- 템플릿 리터럴(``) 사용 금지 → 문자열 결합(+) 사용
```

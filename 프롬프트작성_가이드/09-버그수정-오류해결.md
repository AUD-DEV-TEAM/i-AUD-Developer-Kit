# 버그 수정 / 오류 해결 프롬프트 예제

> 보고서 실행 중 발생한 오류를 분석하고 수정할 때 사용합니다.

---

## 핵심 포인트

- **오류 메시지 전문**을 그대로 포함 (스택 트레이스 포함)
- **재현 절차**를 단계별로 설명
- **기대 동작 vs 실제 동작**을 구분
- 관련 **파일 경로**와 **컨트롤 정보** 명시
- 이전 수정 이력(git commit)이 있으면 포함

---

## 예제 1: 런타임 오류 수정

```
보고서 실행 시 오류가 발생합니다. 수정해줘.

<보고서>
src/reports/Work/OrderMgmt/

<오류 메시지>
TypeError: Cannot read properties of undefined (reading 'GetValue')
  at OrderMgmt.script.ts:45

<재현 절차>
1. 보고서 실행
2. BTN_SEARCH 클릭
3. 조회 완료 후 GRD_MAIN 첫 번째 행 클릭
4. 오류 발생

<기대 동작>
행 클릭 시 TXT_ORDER_ID에 주문번호가 표시되어야 함

<실제 동작>
콘솔에 TypeError 발생, 화면 반응 없음

OrderMgmt.script.ts 를 읽고 45번째 줄 근처의 문제를 분석해줘
```

## 예제 2: 서버 스크립트 오류

```
서버 스크립트 실행 시 오류가 발생합니다.

<보고서>
src/reports/Work/OrderMgmt/

<오류 메시지>
[ServerScript Error] SaveOrders.ts
javax.script.ScriptException: TypeError: Cannot find function from in object ...
  at line 23

<원인 추정>
- 서버 스크립트에서 Array.from()을 사용한 것 같음
- Rhino 엔진은 ES5만 지원하므로 Array.from() 사용 불가

ServerScript/SaveOrders.ts 를 읽고 Rhino 호환되지 않는 코드를 모두 수정해줘
Rhino 제약사항: Array.from, Object.entries, Map, Set, Promise, 화살표 함수, in 연산자 등 사용 불가
```

## 예제 3: 레이아웃/UI 오류

```
보고서 화면이 의도대로 표시되지 않습니다.

<보고서>
src/reports/Work/Dashboard/

<현재 상태>
- GRD_MAIN이 화면 우측에 잘려서 표시됨
- 브라우저 크기를 줄이면 GRD_MAIN이 보이지 않음

<기대 동작>
- GRD_MAIN이 화면 전체 너비에 맞게 표시
- 브라우저 크기 변경 시에도 자동 리사이즈

<관련 설정>
.design.json에서 GRD_MAIN의 Position과 Docking 설정을 확인해줘.
좌우 도킹이 빠져있으면 추가하고, 필요하면 Margin도 조정해줘.
```

## 예제 4: 데이터 바인딩 오류

```
조회는 되는데 그리드에 데이터가 표시되지 않습니다.

<보고서>
src/reports/Work/EmpMgmt/

<재현 절차>
1. BTN_SEARCH 클릭 → 서버 호출 성공 (콘솔에서 확인)
2. 서버 응답에 데이터 10건 포함 (네트워크 탭에서 확인)
3. GRD_EMP에 빈 행만 표시됨

<의심 사항>
- DataSource 이름과 그리드 바인딩 DataSource가 불일치할 수 있음
- 컬럼 Name이 SQL 결과 컬럼명과 다를 수 있음

.design.json의 DataSource 바인딩과 GRD_EMP의 Columns.Name을 확인해줘.
SQL 결과 컬럼명: EMP_NO, EMP_NAME, DEPT_CD, SALARY, HIRE_DT
```

## 예제 5: 기존 수정 이력 기반 오류 분석

```
이전에 수정한 기능에서 사이드 이펙트가 발생한 것 같습니다.

<오류 내용>
GroupGrid에서 총합계가 상단으로 배치되었고, 소계를 숨김 옵션으로 했을 시
하위 첫번째 항목의 텍스트가 화면에 보이지 않음

<기존 수정 이력 git history>
커밋: 7c8a8db - (BCRM#F6023) DataGrid에서 Group 설정하면 데이터가 2줄로 나타나는 현상
커밋: 6fbfac9 - (BCRM#F6023) 동일 건 추가 수정

<그리드 설정 정보>
(GroupGrid의 .design.json 관련 설정을 여기에 붙여넣기)

위 git 이력의 변경 내용을 확인하고, 이번 오류와의 관련성을 분석해줘.
```

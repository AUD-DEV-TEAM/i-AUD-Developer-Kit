# 기존 보고서 수정 / 기능 추가 프롬프트 예제

> 이미 만들어진 보고서에 기능을 추가하거나 동작을 변경할 때 사용합니다.

---

## 핵심 포인트

- **기존 코드를 먼저 읽도록 요청** — AI가 현재 구조를 파악한 후 수정
- **변경 범위**를 명확히 (어떤 파일의 어떤 부분)
- **기존 동작 유지** 조건이 있으면 명시
- 가능하면 **변경 전/후 동작**을 설명

---

## 예제 1: 기존 보고서에 컨트롤 추가

```
src/reports/Work/OrderMgmt/ 보고서에 엑셀 내보내기 기능을 추가해줘.

<현재 상태>
- 상단 툴바에 BTN_SEARCH, BTN_SAVE 버튼이 있음
- GRD_MAIN (DataGrid)에 주문 데이터 표시

<추가할 내용>
1. .design.json에 BTN_EXPORT (Button) 추가: "엑셀 다운로드", BTN_SAVE 우측에 배치
2. OrderMgmt.script.ts에 BTN_EXPORT 클릭 이벤트 추가:
   - GRD_MAIN 데이터를 Excel로 내보내기
   - Matrix.ExportExcel 또는 그리드의 Export 메서드 사용

먼저 현재 .design.json과 .script.ts를 읽고 기존 구조에 맞게 추가해줘
```

## 예제 2: 기존 이벤트 로직 수정

```
src/reports/Work/OrderMgmt/OrderMgmt.script.ts 의
BTN_SAVE 클릭 로직을 수정해줘.

<현재 동작>
- BTN_SAVE 클릭 시 바로 서버에 저장 요청

<변경할 동작>
- BTN_SAVE 클릭 시 먼저 확인 대화상자 표시 ("저장하시겠습니까?")
- "확인" 선택 시에만 저장 진행
- 변경된 행이 없으면 "변경된 데이터가 없습니다" 알림 후 리턴

기존 저장 로직은 그대로 유지하고 앞에 검증만 추가해줘
```

## 예제 3: SQL 조건 추가

```
src/reports/Work/OrderMgmt/DataSource/DS_ORDERS.sql 을 수정해줘.

<현재 SQL>
- ORDER_DATE 범위와 KEYWORD 검색 조건만 있음

<추가할 조건>
- STATUS 필터 추가: CMB_STATUS (ComboBox)에서 선택한 상태값으로 필터
- DEPT_CODE 필터 추가: CMB_DEPT (ComboBox)에서 선택한 부서코드로 필터
- 두 조건 모두 비어있으면 조건 무시 (@AND 패턴)

현재 SQL을 먼저 읽고 기존 구조에 맞게 조건을 추가해줘
/iaud-sql-guide 참조
```

## 예제 4: 그리드 컬럼 추가/변경

```
src/reports/Work/OrderMgmt/ 보고서의 GRD_MAIN (DataGrid)을 수정해줘.

<추가할 컬럼>
1. PRIORITY 컬럼 추가: 콤보박스 (높음/중간/낮음)
2. ASSIGNEE 컬럼 추가: 텍스트, 편집 가능
3. DUE_DATE 컬럼 추가: 날짜, Calendar 컬럼

<변경할 컬럼>
- AMOUNT 컬럼: 포맷을 "#,##0" → "#,##0.00" 으로 변경
- STATUS 컬럼: 너비 80px → 120px 로 변경

.design.json의 GRD_MAIN Columns 배열을 수정해줘
```

## 예제 5: 서버 스크립트 로직 추가

```
src/reports/Work/OrderMgmt/ServerScript/SaveOrders.ts 를 수정해줘.

<현재 동작>
- 단순 INSERT/UPDATE/DELETE 처리

<추가할 로직>
- 저장 전 유효성 검증:
  - AMOUNT가 0 이하이면 오류 반환
  - ORDER_NAME이 빈 값이면 오류 반환
  - 중복 ORDER_ID 체크 (INSERT 시)
- 저장 성공 후 감사 로그 테이블(TB_AUDIT_LOG)에 기록

현재 코드를 먼저 읽고 기존 패턴에 맞게 로직을 추가해줘
/iaud-server-script 참조
```

## 예제 6: TypeScript 전환 (JS → TS)

```
src/reports/Work/OldReport/OldReport.script.js 를 TypeScript로 전환해줘.

<전환 규칙>
- var → let/const 변환 (TDZ 주의)
- Matrix.getObject() 결과에 구체적 타입 캐스트 추가
- 이벤트 핸들러 파라미터에 타입 어노테이션 추가
- Constructor 함수는 class로 변환하지 않음
- as any 사용 금지

/iaud-ts-conversion 참조
```

## 기존 보고서 수정 시 주의사항

```
기존 보고서를 수정할 때 다음 사항에 주의해줘:

1. 수정 전에 반드시 현재 파일을 먼저 읽어서 기존 구조 파악
2. 기존 이벤트 핸들러나 함수를 덮어쓰지 않기
3. 기존 변수명/함수명 컨벤션을 유지
4. .design.json 수정 시 ID 값을 변경하지 않기
5. 불필요한 import 추가하지 않기
6. 기존에 동작하던 기능이 깨지지 않도록 주의
```
